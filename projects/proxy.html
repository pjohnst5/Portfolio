About

This was a semester long project in which I developed a multi-threaded proxy server complete with signal handling and semaphores to avoid race conditions. It concurrently served multiple HTTP requests while also caching the most recent responses. The cache was managed through the use of semaphores to write-lock it when one thread was reading from it and read-lock it when a thread was writing to it. I used Malloc and Free to allocate and deallocate the memory which the cache used and I also kept track of where to write in the cache with head and tail pointers. In addition, I used queues to track which requests had come most recently and opened sockets to communicate with requested servers.
